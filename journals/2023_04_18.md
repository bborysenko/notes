- ## [[How to exclude some page from graph view in Logseq?]]
	- To exclude specific page from graph view in Logseq set the page property `exclude-from-graph-view:: true`.
- ## [[How to stop Logseq to create properties pages?]]
	- By default Logseq creates own pages for each page properties. You may feel that it adds some clutter to your graph. Thankfully there is a [two options](https://docs.logseq.com/#/page/term%2Fproperties) within `config.edn` you can use. Setting `:property-pages/enabled?` option to `false` disable creating pages at all.
	- Or, if you need to keep some property pages, you can exclude specific properties with `:property-pages/exclude-list`.
- ## [[How do I use Logseq for my Digital Garden?]]
	- To make the graph view clearer, I did following:
		- Excluded "type" pages from the graph view such as [[Article]], [[Book]], [[Person]], [[Tool]] by setting the page property `exclude-from-graph-view:: true`.
		- Excluded pages created by the ChatGPT plugin like [[user]] and [[assistant]].
		- Disabled property page creation by setting the `:property-pages/enabled?` option to `false`.
