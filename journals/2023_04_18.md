- ## [[How to exclude some page from graph view in Logseq?]]
	- To exclude specific page from graph view in Logseq set the page property `exclude-from-graph-view:: true`.

- ## [[How Terragrunt can improve Terraform code layering?]]
	- When dealing with large Terraform deployments, it's often helpful to split them into smaller, more manageable parts. However, this approach can lead to additional work and repetition, as each subdirectory requires its own `provider.tf` and `backend.tf` files, which must be edited individually. Additionally, running Terraform for each subdirectory separately can add to the workload.
	- Fortunately, Terragrunt provides a solution to these issues. Instead of manually preparing files for each subdirectory and then calling Terraform multiple times, you can define Terragrunt configuration files and call Terragrunt once in the root directory. Terragrunt then performs all necessary actions recursively, eliminating the need for repetitive tasks.
	- While Terragrunt runs Terraform in parallel by default, this can sometimes result in errors if different layers depend on one another. To avoid this issue, you can instruct Terragrunt to run Terraform in sequence by defining a dependency for each subdirectory in the Terragrunt declaration file. This ensures that each layer is executed in the correct order, preventing any potential errors.
